# Testing Reference

Full test-file catalog and the embedded-assertion (`hover:`/`def:`/`diag:`/…) format. For the testing *workflow* (commands, `--workspace`, default-off-code rules, the "don't write `diag: none`" rule), see [CLAUDE.md — Testing](../CLAUDE.md#testing). When fixing a bug, add a regression test to the appropriate file below using the annotation format — see [CLAUDE.md — Bug fixes](../CLAUDE.md#bug-fixes).

## Test file layout
- `tests/integration_test.rs` — Unified test harness with `TestConfig`
- `tests/integration.lua` — Basic hover/def: primitives, functions, scopes, varargs, addon namespace
- `tests/integration_stubs.lua` — Stubs hover/def: external globals, Frame type
- `tests/annotations.lua` — Annotation features: @param, @return, @type, @class, @field, @alias
- `tests/overloads.lua` — Overload resolution (--with-stubs)
- `tests/or-polyfill.lua` — Version-polyfill idiom `local f = _G.SomeAPI or function(...) end`: a `function or function` `or` decays to bare `function` (callable but arity/param unchecked) so calls written for the broader/different-arity fallback don't false-flag `type-mismatch`/`redundant-parameter`/`missing-parameter`; negative controls verify a direct stub call and a `function or <non-function>` stay checked (--with-stubs)
- `tests/deep-inheritance.lua` — 5-level class hierarchy (--with-stubs)
- `tests/signature-help.lua` — Signature help with `sig:` assertions (--with-stubs)
- `tests/diagnostics/` — Semantic diagnostics with `diag:` assertions and @diagnostic suppression; `.wowluarc.json` enables `need-check-nil` + `implicit-nil-return`. Includes `invalid_op.lua` for `invalid-op` (arithmetic/concatenation on incompatible types) and `nil_table_key.lua` for `nil-table-key` (nil in table key type annotations)
- `tests/need-check-nil/` — Nil-checking diagnostics with nil-guard narrowing; `.wowluarc.json` enables the default-off `need-check-nil` code
- `tests/nil-index.lua` — Nil table key diagnostics (`nil-index`): read/write nil keys, narrowing suppression, literal keys, key type inference nil-stripping
- `tests/access-modifiers/` — Private/protected field access diagnostics; `.wowluarc.json` enables `inference.implicit_protected_prefix`
- `tests/references.lua` — Find references and rename
- `tests/references-globals.lua` — Find-references regressions for collisions with real WoW globals: a method name that doubles as a `_G.X` global must resolve to the method (not pull in unrelated `:X()` calls), and `_G.X` / `local g = _G; g.X` must fall through to symbol lookup so find-references still works on explicit global accesses (--with-stubs)
- `tests/undefined-global.lua` — Undefined global diagnostics (--with-stubs)
- `tests/undefined-field.lua` — Undefined field on @class tables diagnostics
- `tests/undefined-doc-class.lua` — Undefined class names in `@class Foo: Parent` inheritance position, invalid class parents (primitive types, string literals, union types)
- `tests/undefined-doc-name.lua` — Undefined type names in annotations (`@param`, `@return`, `@type`, `@field`, `@alias`, fun()/inline table shapes)
- `tests/circle-doc-class.lua` — Circular @class inheritance chain diagnostics
- `tests/class-shadows-builtin.lua` — Workspace `@class` (with explicit `@field`s) reusing a built-in WoW API class name (`class-shadows-builtin`), `@diagnostic` suppression, and verifying the per-file local shape does not inherit stub fields (--with-stubs)
- `tests/class-shadows-crossfile/` — Cross-file record collision: a workspace `@class` reusing a stub name with its own `@field`s, used as a `@type table<K, Class>` field on a `@class` table (resolved against the *external*, additively-merged class), so `missing-fields` must scope the required set to the workspace's declared fields (no false positive). Two files declare the class to exercise additive partial-class merging. Uses `scan_dir` + `--with-stubs`
- `tests/class-shadows-library/` — Additive merge safety: a `library` file reusing a stub class name merges additively (the stub table is never replaced), so the built-in `Frame` keeps its methods (`SetPoint`) workspace-wide and the library's field is visible on top — a suppressed-origin file can't strip a built-in. Uses `scan_dir` + `--with-stubs`
- `tests/generics.lua` — Generic type parameters with `@generic`, plus `generic-constraint-mismatch` for class constraints and for constrained parameterized aliases (`@alias Foo<T: Constraint>` enforced at `@type`/`@param`/`@return`/`@field` use sites)
- `tests/generics-projections.lua` — `params<F>` and `returns<F>` utility-type projections with generic registry pattern
- `tests/generics-projections-e2e.lua` — End-to-end generic registry class exercising `params<F>` and `returns<F>` through class fields and table constructors
- `tests/call-func-generics.lua` — Class type parameter substitution into `@overload` resolution and `returns<F>` projections for callable tables and for-in loops; also `@requires T: Constraint` method gating (`param-constraint-mismatch`) and `@return self<X>` re-parameterized return
- `tests/requires-misuse.lua` — Misuse of `@requires`/`@return self<X>`: not attached to a function, naming a non-class-type-param, malformed shape, undefined constraint type, non-generic class, and wrong self<X> arity (`doc-func-no-function`/`malformed-annotation`/`undefined-doc-name`), plus the valid case
- `tests/funcall-access.lua` — Dot/colon access on function call return values
- `tests/builder-pattern/` — `@builds-field` and `@return built` builder pattern with edge cases and diagnostics; `.wowluarc.json` enables `need-check-nil`
- `tests/return-overloads.lua` — Tuple-union `@return` (`(A, B) | (C, D)`) sibling narrowing and variadic return expansion (`@return ...T`)
- `tests/tuple-union-returns.lua` — Focused tuple-union coverage: single-tuple shorthand, labels, per-case descriptions, `fun()` and `@alias` propagation, mixing/arity diagnostics
- `tests/multi-return-inference.lua` — Body-derived return-arity inference for functions with **no** `@return` annotation: the inferred arity must reflect the widest arity across all return statements (not collapse to the first/single value), so destructuring a multi-return to its exact arity emits no `unbalanced-assignments`. Covers mixed value types, multi-branch same/differing arity (widest-in-middle), early bare `return`, correlated set-or-nil, single-value preservation, and the open-ended vararg/tail-call cases; plus the genuine over-destructure / single-value true positives
- `tests/multi-return-inference-crossfile/` — Cross-file counterpart exercising the deferred body-return harvest (`analysis/deferred.rs`): `defs.lua` declares unannotated multi-return globals + an addon-ns method, `user.lua` destructures them in another file; exact-arity destructures stay clean, over-destructure still warns. Uses `scan_dir` + `--with-stubs`
- `tests/cast.lua` — `@cast` (replace/add/remove) and `@as` inline expression type assertions
- `tests/annotation-completion.lua` — Annotation doc-comment completions: tag names, `@param` names, type suggestions
- `tests/string-literal-completion.lua` — String literal completions in `==`/`~=` comparisons against string literal union types: field access, simple variables, method call returns, single-quote, partial typed, nested field access
- `tests/backtick-generic-completion.lua` — Class-name completions for backtick-generic params (`@param p `T``): constrained `@generic T: Base` scopes to subclasses of Base, unconstrained `@generic T` offers every class, colon-method call param-index offset, partial-prefix
- `tests/event-hover/` — Event payload hover via `@event` annotation: multi-param line-breaking, single-param inline, empty payload, custom event types; uses `scan_dir` to load event declarations from `events.lua`
- `tests/callback-events/` — `@generates-events` synthesized `.Event` enum table on a `CallbackRegistryMixin` receiver: `defs.lua`/`user.lua` cover same-file + cross-file `.Event` access (string-literal and field-reference entries); `synth.lua` is a **workspace-only** mixin (custom events the stubs can't know) that genuinely exercises the synthesis path in the harness. Uses `scan_dir` + `--with-stubs`
- `tests/callback-registry/` — `@callback-event-arg` event-name completion + `unknown-callback-event` diagnostic for the addon pattern (`addonTable.CallbackRegistry:RegisterCallback("…")`): `lib.lua` declares the producer/consumer methods in-workspace, `constants.lua` the `addonTable.Constants.Events` string array, `init.lua` the `GenerateCallbackEvents(addonTable.Constants.Events)` registry declaration, `user.lua` the cross-file `comp:`/`diag:` assertions. `.wowluarc.json` enables the default-off code; uses `scan_dir` + `--with-stubs`
- `tests/callback-multi-addon/` — proves per-addon registry scoping: `AddonA`/`AddonB` each `addon_root: true` declare their own `addonTable.CallbackRegistry` with different events; `AddonA/user.lua` asserts AddonB's event is `unknown-callback-event` in AddonA (would pass spuriously without addon-scoped keys). Shared `lib.lua` (workspace mixin defs) at the scan root. The incremental-refresh path is covered by the `maybe_rebuild_workspace_refreshes_callback_registries` unit test in `main_loop/mod.rs`
- `tests/ace3/` — bundled Ace3 library typing: `addon.lua` asserts the `---@class Foo : AceAddon-3.0` convention resolves the embeddable `AceAddon` instance methods (`NewModule`/`GetModule`/`GetName`/…) plus the `AceConsole-3.0`/`AceEvent-3.0` parent names (no `undefined-field`/`undefined-doc-class`); `module.lua` asserts the `---@class Foo : AceModule` convention (the type of `Addon:NewModule(name)`/`GetModule(name)`) resolves and inherits the `AceAddon` instance methods (`AceModule : AceAddon`, defined in `stubs/overrides/AceAddon-3.0.lua`); `locale.lua` asserts default-locale `NewLocale(app, locale, true [, silent])` returns a non-nil table (the `L[key] = true` idiom is clean) while a non-default `NewLocale` stays nilable (`need-check-nil` on an unchecked write). `.wowluarc.json` enables `need-check-nil`; uses `scan_dir` + `--with-stubs`. The AceAddon-3.0/AceLocale-3.0 definitions live in `stubs/overrides/`; AceConsole/AceEvent/AceHook come from Ketho's vendored Ace3 annotations
- `tests/expression-type/` — `expression<C, R>` type: hover/completions/definition inside expression strings, `undefined-field` and `type-mismatch` diagnostics, return type inference, inherited fields, generic-`R` inference (R inferred from the expression body and propagated to `@return Schema<R>`), and the incomplete-context suppression (an unbound generic-param member of `C` suppresses `undefined-field` instead of false-positiving)
- `tests/expression-deferred-harvest/` — cross-file deferred type-arg harvest: a defclass `__init` self-field assigned a chained generic call is typed `any` by the coarse scan; the harvest recovers `Mgr<State>` so `expression<T & Builtins, R>` resolves the bound state's fields (`user.lua`). `builder_module_user.lua` covers the related cache-discipline case — a module-table field (not a defclass self-field) assigned `Mgr.Create(state)` where `state` is a `@return built: Base` builder result, whose generic refines base→subtype late in the fixpoint and must not be cached at the stale base. Uses `scan_dir` with `lib.lua` (framework) + the two user files; needs `--with-stubs` (the harvest map is populated on the `build_on_stubs` path)
- `tests/call-hierarchy.lua` — Call hierarchy queries: `call_hierarchy_item_at` (functions and methods), `outgoing_calls_from_function` (grouped call ranges, nested function exclusion), `call_sites_for_function` (incoming call sites with enclosing function), `enclosing_function_at`, `call_hierarchy_display_name` (method vs function formatting)
- `tests/code-lens.lua` — Code lens assertions via `lens:` field: top-level functions, local functions, class methods (colon syntax), table functions (dot syntax), "N implementations" on `@class` declarations counting direct subclasses, "overrides Parent" on methods overriding parent class methods, multi-level inheritance (grandparent override resolution)
- `tests/type-definition.lua` — Go to Type Definition assertions via `typedef:` field: variables typed as `@class` navigate to the class declaration, `@alias (opaque)` navigates to alias, union types navigate to first class member, field access on class tables navigates to field's type, primitives return None
- `tests/type-narrows.lua` — `@type-narrows` custom type guard narrowing (then-branch, early-exit, else-branch, assert, method-style)
- `tests/isobjecttype-narrows.lua` — IsObjectType() narrowing via `@type-narrows` on FrameScriptObject: then-branch, early-exit, and-chain, else-branch no-narrow, non-literal/unknown class graceful degradation, non-Button subclass (--with-stubs)
- `tests/getobjecttype-narrows.lua` — `GetObjectType() == "ClassName"` narrowing via `@returns-class-name` on FrameScriptObject (the return-value-comparison counterpart to `@type-narrows`): then-branch, guarded-access resolves (the motivating false positive), reversed operands, `~=` early-exit, and-chain, assert, return/param type match, else-branch no-narrow, non-literal/unknown-class graceful degradation, unguarded-is-error true positive, and a local `@returns-class-name` method whose trailing tokens are ignored (no greedy `@return` misparse) (--with-stubs)
- `tests/type-guard.lua` — `type()` guard narrowing for symbols and field chains (`type(x) == "string"`, `type(obj.field) == "table"`, `type(x) ~= "nil"`)
- `tests/union-field-narrow.lua` — Union member narrowing based on field presence: `if info.title then` narrows union to members where `title` is required (if/else, early exit, assert, nil comparison, three-way unions, optional field discrimination)
- `tests/literal-bool-ret.lua` — Literal boolean return type union discrimination (`@return true`/`@return false` on union member methods)
- `tests/correlated-locals.lua` — Correlated local variable narrowing: locals assigned in every branch of if/elseif (no else) are narrowed together
- `tests/lateinit/` — `T!` non-nil assertion / lateinit fields: `@field` and `---@type` with `!` suffix; `.wowluarc.json` enables `need-check-nil`
- `tests/count-down-loop.lua` — Numeric for-loop step direction diagnostics (`count-down-loop`)
- `tests/incomplete-signature-doc/` / `tests/incomplete-signature-doc-meta/` — `incomplete-signature-doc` HINT for functions with partial `@param`/`@return` annotations; `-meta` asserts `@meta` files suppress the diagnostic. Each dir has a `.wowluarc.json` enabling the default-off code.
- `tests/missing-annotation/` — `missing-param-annotation`/`missing-return-annotation` HINTs. `test.lua`: globals and table methods/fields missing `@param`/`@return` are flagged (even with zero annotations); `local function`, anonymous literals, and forward-declared-local rebindings are not. `escape.lua`: cross-file table-escape scoping — methods on a local table are flagged only when the table escapes (global, `@class`, or attached to the addon namespace), not when it's file-private. `.wowluarc.json` enables the default-off codes.
- `tests/stylistic.lua` — Stylistic HINT diagnostics: `empty-block`, `redundant-return`, `trailing-space`
- `tests/not-precedence.lua` — Operator precedence: `not x <cmp> y` parses as `(not x) <cmp> y` (`not-precedence`)
- `tests/redundant-logical.lua` — `redundant-or` (truthy LHS makes `or` RHS dead code), `redundant-and` (falsy LHS makes `and` RHS dead code, or always-truthy LHS makes `and` a no-op outside the `x and y or z` ternary idiom) HINT diagnostics, `@diagnostic` suppression
- `tests/redundant-condition/` — `redundant-condition` (always-truthy/falsy `if`/`elseif`/`while` conditions) HINT diagnostic; `.wowluarc.json` enables the default-disabled code
- `tests/syntax-coverage.lua` — Broad syntax construct coverage: numeric literals, long strings, unary/binary operators, repeat/until, for-step, semicolons, no-paren calls, anonymous functions, multi-dot definitions, code-after-break, long bracket comments, forward-declared locals, nested function returns, bracket-keyed tables, multi-target assignment, conditional function defs, higher-order functions, module patterns, closures, reassignment, colon methods
- `tests/convergence.lua` — Fixpoint convergence regression: 60 reverse-order function calls testing inner loop optimization
- `tests/metatable-type-i.lua` — Metatable type inference: `setmetatable()` + `__index` field propagation, chained metatables, self-referential `mt.__index = mt`, factory functions, instance field priority (--with-stubs)
- `tests/infinite-loop.lua` — Infinite loop handling (`while true` / `repeat until false`): only branching returns produce confident non-nilable return types and suppress `missing-return`
- `tests/opaque-alias.lua` — `@alias (opaque)` nominally distinct type aliases: hover display, literal assignability (Rule 2), cross-alias errors, outward flow to base type, arithmetic decay, opaque in unions, opaque as return type, comparison and concatenation
- `tests/overlay.lua` — Per-file overlay where fields are added to class-typed local variables (runtime field additions)
- `tests/structural-subtype.lua` — Structural subtyping: table literals assignable to `@class` types when field shapes match
- `tests/mixin-data-params.lua` — mixin-object params accept the data type, methods-typed params are strict: a plain data table to a WoW C-API color/itemlocation param is clean, garbage (`{foo}`) is `missing-fields`, a real `CreateColor()` object is accepted, and a plain `{r,g,b}` table to a user `@param colorRGBA` (the methods type) is `missing-fields` (--with-stubs)
- `tests/array-element-supertype.lua` — array element-type common-supertype inference: a constructor whose element type is a most-specific intersection (`Frame & InsetFrameTemplate`) must not pin the array so strictly that a later `table.insert` of a less-specific sibling (a plain `Frame`, a `Button`) false-positives `type-mismatch`. The element type checked at the insert is widened to the union of the intersection's facets (a common supertype), while a genuinely-unrelated value (string/number) is still flagged. Reads keep the precise intersection (`ipairs` loop var), and a hand-written `@param p A & B` (not an array-bound generic) stays strict as a negative control (--with-stubs)
- `tests/accessor-modifiers.lua` — `@accessor` annotation for transparent access modifier fields (private/protected through accessor methods)
- `tests/semantic-tokens.lua` — Semantic-token classification via the `tok:` assertion: function/method/class/namespace/parameter/property/variable tokens with `defaultLibrary`/`deprecated` modifiers (--with-stubs)
- `tests/inlay-hints/` — Inlay hint assertions via `hint:` field: parameter names, variable types, return types, for-loop types, parameter types (disabled by default, enabled in test config), suppression cases (name match, annotated, nil, function RHS, void function, multi-assignment, pairs/ipairs, self param, Any param); `.wowluarc.json` enables all hint categories
- `tests/backward-inference.lua` — Backward param-type inference signals: arithmetic/unary/concat, typed-argument propagation, annotated-param precedence, conflict fallback, overload-aware arity selection (2-arg call must pick the 2-arg `@overload`, not the 3-arg primary)
- `tests/backward-inference-disabled/` — Verifies `inference.backward_param_types: false` in `.wowluarc.json` disables the inference pass
- `tests/correlated-return-inference/` — Synthesized correlated return-only overloads (default-on; explicit `inference.correlated_return_overloads: true`): basic 2-tuple narrowing, 3-tuple, early-exit, skip cases (existing `@return`, single return, mixed tuples, all-nil only, arity 1), mismatched-arity padding (shorter returns padded with nil up to max arity)
- `tests/correlated-return-inference-disabled/` — Verifies `inference.correlated_return_overloads: false` disables synthesis: nested-scope returns leave callers with `?`
- `tests/correlated-return-inference-disabled-crossfile/` — Cross-file global function with synthesizable return pattern, verifying workspace-scan synthesis path honors the `correlated_return_overloads: false` flag
- `tests/allowed-globals/` — Allowed globals via `.wowluarc.json` config (`globals.read`/`globals.write`), `create-global` diagnostic, `SLASH_*` auto-detection (default `allow_slash_commands: true`), and `BINDING_HEADER_*`/`BINDING_NAME_*` auto-detection (default `allow_binding_globals: true`). Also covers that a top-level field/index write through a parenthesized/prefix-expression base (`(A or B).field = v`, `(getObj()).f`, `({}).f`, plus bracket-key and deeper-chain forms) is not a bare-name global (no `create-global`) — the `Identifier::has_prefix_expr_base()` guard in `build_ir.rs` + `scan_globals.rs` main loop
- `tests/create-global-prefix-crossfile/` — Cross-file counterpart pinning the `scan_globals.rs` *descendants-pass* `has_prefix_expr_base` guard, which the top-level cases above never reach: `defs.lua` writes fields through a parenthesized prefix inside a function body and in a top-level multi-target assignment (the two shapes only the descendants pass scans); `user.lua` reads the names bare and asserts they stay `undefined-global` (the guard kept them from registering as phantom globals). Uses `scan_dir` + `--with-stubs`
- `tests/slash-commands-disabled/` — Verifies `globals.allow_slash_commands: false` makes `SLASH_*` globals trigger `create-global` and `undefined-global`
- `tests/binding-globals-disabled/` — Verifies `globals.allow_binding_globals: false` makes `BINDING_HEADER_*`/`BINDING_NAME_*` globals trigger `create-global` and `undefined-global`
- `tests/dynamic-global-prefix/` — Dynamic global prefix/suffix detection: `_G["PREFIX"..k]` and `_G[name.."SUFFIX"]` patterns auto-register wildcard allowed globals so reads don't false-positive as `undefined-global`
- `tests/saved-variables/` — `.toc` file `SavedVariables`/`SavedVariablesPerCharacter` auto-discovered as allowed globals; multiple `.toc` files in one directory
- `tests/unused-function/` — Cross-file `unused-function` diagnostic for global functions and table methods; uses `scan_dir` with `defs.lua` (definitions) and `user.lua` (references)
- `tests/unused-vararg/` — `unused-vararg` diagnostic for functions declaring `...` but never referencing it; uses `.wowluarc.json` to enable the default-disabled code
- `tests/unknown-types/` — Strict-typing `unknown-param-type` / `unknown-return-type` / `unknown-local-type` / `unknown-field-type` diagnostics; uses `.wowluarc.json` to enable the four default-disabled codes
- `tests/flavor-filter/` — Flavor filtering via `.wowluarc.json` (`flavors`), `@flavor-narrows` annotation, `WOW_PROJECT_ID` narrowing, TOC-based per-file flavor detection, and the `wrong-flavor-api` diagnostic. One subdirectory per scenario (classic-only, multi-flavor, wow-project-guard, annotation-guard, boolean-guard, boolean-guard-crossfile, invalid-annotation, no-config, suppression, toc-suffix, toc-per-line, toc-intersect, toc-header-restrict).
- `tests/deprecated-flavor/` — Flavor-aware `deprecated` (`crate::flavor::deprecation_suppressed`): a retail-sourced `@deprecated` API still live on a flavor the addon targets isn't flagged. One subdir per scenario, all `--with-stubs`: `retail-config` (config flavors=retail → warns, and the `tok:` assertion confirms the semantic token carries the `deprecated` modifier), `classic-config` (config flavors=classic_era → suppressed; `tok: function defaultLibrary` assertions confirm the token is **not** struck through, mirroring the absent warning), `multi-flavor` (config retail+era → `deprecated` suppressed but `wrong-flavor-api` still fires on `GetMerchantItemInfo`, proving the two are independent), `toc-interface` / `toc-multi-version` / `toc-retail` (no config — flavor breadth comes from the `.toc` `## Interface:` line; multi-version proves `wrong-flavor-api` stays OFF since the Interface line doesn't feed `project_flavors`), `no-flavor` (no config + no `.toc` → `addon_flavors == 0` → warns as before), `workspace-deprecated` / `workspace-deprecated-crossfile` (a workspace-authored `@deprecated` always warns — the suppression is `is_stub_function`-gated).
- `tests/framexml-disabled/` — Verifies `framexml: false` in `.wowluarc.json` disables FrameXML globals while keeping core WoW API globals
- `tests/xml-frames/` — XML frame/template scanning: virtual templates create classes, non-virtual frames create globals, `parentKey`/`parentArray` child fields, `KeyValue` typed fields, `inherits` template inheritance, `$parent` name resolution, intrinsic elements, implicit parentKey on special elements; uses `scan_dir` with `frames.xml` + `user.lua`
- `tests/self-field-mixin/` — Cross-file self-field tracking on a *mixin* table: a plain global `DataProviderMixin = {}` (no Lua `@class`) promoted to a class only by `widgets.xml`'s `mixin="DataProviderMixin"`. Its `self.field = ...` writes in one file (`provider.lua`) — a bare value, a function literal (registered callable, no `cannot-call`), and a chained funcall (existence-only) — must all resolve when read from another method on the same mixin in `consumer.lua`. Exercises the `scan_globals.rs` descendants-pass `self` handler that the typed/funcall/bare self-field scanners can't reach. Uses `scan_dir` + `--with-stubs`.
- `tests/mixin-self-field/` — `@narrows-arg` mixin applied to a *field* target (`Mixin(self.Child, M)` after `self.Child = CreateFrame(...)`), the cross-method counterpart to the plain-local `Mixin(f, M)`. `host.lua` is an addon-ns mixin table (no `@class`) whose `self.field` is typed per-file — the field's registered type must carry the intersection so the mixin methods resolve from a *sibling* method (`OnLoad` writes + mixes, `Layout` reads), covering multiple mixins, a mixin nested in control flow, and a negative control (a non-mixin'd field still reports unknown methods). `classhost.lua` is the same shape on a `@class` mixin. `reader.lua` covers `Mixin(tbl.field, M)` on a function-local table. Uses `scan_dir` + `--with-stubs`.
- `tests/self-field-mixin-collision/` — Regression for the descendants-pass `self` handler's leaf-name keying, covering all three legs of its gate: (1) a *deeply-nested* receiver whose leaf is NOT a known `@class` (`function MyAddon.B.Collision:Method()`) collapses to its leaf (`Collision`), which must NOT leak the self-field onto an unrelated top-level global of the same name (`Collision = {}`) — skipped; (2) a *single-name* mixin (`function PlainMixin:Init()`, the method's own root) still resolves cross-file; (3) a *deeply-nested* receiver whose leaf DOES name a known `@class` (`function Outer.Sub.Widget:Configure()` with `---@class NestedClass`) re-keys to the class name via `var_to_class`, and its `self.nested` write is nested **inside control flow** so it routes *exclusively* through the descendants pass (the typed/bare scanners only catch top-level method-body writes), then resolves on a `NestedClass`-typed value in `user.lua`. Uses `scan_dir` + `--with-stubs`.
- `tests/library-dirs/` — `library` config field: relative directory patterns scanned for types but with diagnostics suppressed; `.wowluarc.json` marks `libs/` as library; verifies types flow to user files and library diagnostics are suppressed
- `tests/library-dirs-external/` — Absolute path `library` config: external library directory outside the workspace scanned for types; `.wowluarc.json` generated at test time with absolute path; verifies cross-directory type resolution
- `tests/library-dirs-relative-escape/` — Relative `library` path that escapes the workspace root (`../shared`): resolved against the config dir and scanned as an external directory so its globals/types are pulled in while diagnostics stay suppressed
- `tests/multi-addon/` — Multi-addon workspace namespace isolation via `addon_root: true` in per-addon `.wowluarc.json`; verifies AddonA sees only its own namespace fields and AddonB sees only its own
- `tests/crossfile/` — Cross-file addon namespace resolution, `@defclass` with parameterized parent classes, `@builds-field` builder chains, `@class`/`@type` field access, `@class` inheritance, `@alias` usage, global functions/variables, access modifier diagnostics, typed self-field inheritance (`self_field_lib.lua`/`self_field_user.lua`), deep addon-ns chains of 4+ parts with auto-created intermediate sub-tables (`deep_chain_defs.lua`/`deep_chain_user.lua`/`deep_chain_nonroot.lua`), cross-file constrained-alias bound enforcement (`alias_constraint_defs.lua`/`alias_constraint_user.lua`), and a chained-funcall self-field on a redeclared `@class` registered existence-only (`self_field_chain_lib.lua`/`self_field_chain_user.lua`, the `self.db = LibStub(...):New(...)` pattern)
- `tests/multi-definition/` — Multi-result go-to-definition (`defs:`/`typedefs:`): a global function/variable, `@class`, or `@alias` defined in two files (`defs_a.lua`/`defs_b.lua`) lists every definition site; a single-file global still returns exactly one (`user.lua`). Uses `scan_dir` + `--with-stubs`.

## Annotation format
Test expectations are embedded as comments below code lines:
```lua
local x = 5
--    ^ hover: x: number  def: local
foo(1, "hello")
--  ^ sig: fun(x: number, y: string): boolean
oldFunc()
-- ^ diag: deprecated
local y = mustUse()
-- ^ diag: none
```
Fields are separated by double-space. Supported fields: `hover:`, `def:`, `defs:`, `typedef:`, `typedefs:`, `sig:`, `diag:`, `refs:`, `comp:`, `tok:`, `hint:`, `lens:`.

The `defs:`/`typedefs:` field values are integers: the expected number of distinct sites returned by `definitions_at`/`type_definitions_at` (multi-result go-to-definition / go-to-type-definition for a global or type defined in multiple files). `def:`/`typedef:` assert the *primary* (first) result; `defs:`/`typedefs:` assert the total count.

The `tok:` field value is the semantic-token classification at the caret: the token type followed by zero or more modifiers in any order (e.g. `tok: function defaultLibrary`, `tok: method deprecated`). Use `tok: none` to assert no token is emitted at the caret.

The `lens:` field value is the expected code lens function name on the code line (e.g. `lens: greet`, `lens: sayHello`). Use `lens: none` to assert no code lens target exists at that line.

The test harness applies `ProjectConfigs::disabled_diagnostics_for()` to filter diagnostics — the same path the LSP server uses in `publish_with_config`. Tests that rely on default-off codes (`need-check-nil`, `implicit-nil-return`, `unused-vararg`, `incomplete-signature-doc`, `unknown-*-type`) must live in a subdirectory with an adjacent `.wowluarc.json` that opts in via `diagnostics.enable`. Existing examples: `tests/need-check-nil/`, `tests/incomplete-signature-doc/`, `tests/unused-vararg/`, `tests/unknown-types/`.

**Exhaustive diagnostic checking**: The harness fails on any diagnostic (WARNING, ERROR, or HINT) that is not covered by a `diag:` assertion. This catches diagnostic regressions automatically — if a code change introduces a new unexpected diagnostic, the test fails even without an explicit `diag: none` on that line. Therefore, **do not write `diag: none`** — it is redundant since the absence of an unexpected diagnostic is already verified by the harness. Only use `diag: <code>` to assert that a specific diagnostic IS emitted. Syntax errors are exempt from exhaustive checking since they are structural parse failures whose codes are free-form messages that can't be suppressed via `---@diagnostic disable`. To suppress incidental diagnostics (e.g. `unused-local`, `unused-function`, `shadowed-local`) in test files that aren't testing those specific codes, use `---@diagnostic disable: code1, code2` at the top of the file. For test files that both test a HINT diagnostic AND have incidental instances of the same code, use `---@diagnostic disable:` at the top with `---@diagnostic enable:` / `---@diagnostic disable:` blocks around the tested section.
